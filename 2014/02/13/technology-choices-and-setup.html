Tech Choices & Setup

13 Feb 2014

Game genre check. Available time check. Now what? Well, its time to make few technology choices.

I have decided to stick with html5/javascript for the game because it is very easy to develop, host and play. It will have its own flaws but I am ready to deal with it.

Next choice is what will we use to tweening, rendering, controls and most things that make a game. You roll it completely on your own, use some sort of what is now called a javascript game engine (albeit missing lots of properties of game engines) or just use different collection of libraries to help with each element separately. Check out pretty good js engine/libraries comparison. For the purpose of this project I will be rolling with the impact.js since I already own a license for it, played with it so pretty aware of its benefits & limitations and even contributed some code to it. I'd rather minimize the fight with the game engine and focus on the game itself. There is still plenty of yak shaving.

Alright, now with the development process I am going with that I've done a number of times and am confortable with. Repository is on the github http://github.com/tjbladez/html5_strategy, project itself right now is hosted for free on heroku http://html5-strategy.herokuapp.com. Naming it is something I dont care at this point in time and can always do it later. Although you can play the game by just opening index.html in your browser at this point, I have also provided node based express server for the game. Code itself I will write in coffeescript.

Setup was fairly straight forward. Had a minor problem with heroku static file serving. Deployments are ready both for gh-pages and game itself. Lets get something rendering.
